01
Why Migrate from RDD to DataFrame?

RDDs (Resilient Distributed Datasets) are Spark's foundational abstraction — immutable, fault-tolerant collections distributed across a cluster. DataFrames are RDDs of Row objects with a named schema, and they unlock the Catalyst query optimizer and Tungsten execution engine, making them dramatically faster for most workloads.

⚡ RDD
✗No Catalyst optimizer — manual optimization required
✗No schema enforcement — runtime errors from bad types
✗Java object serialization — high GC pressure
✗No predicate pushdown to data sources
✓Full Python/Scala control for complex logic
✓Unstructured data (binary, custom formats)
🚀 DataFrame
✓Catalyst optimizer rewrites your query plan
✓Schema validation catches issues at parse-time
✓Tungsten off-heap memory — minimal GC overhead
✓Predicate pushdown, partition pruning, columnar I/O
✓SQL compatibility — mix DataFrames and Spark SQL
✓Delta Lake, Unity Catalog, Photon engine ready
💡
The Golden Rule
Stay in DataFrame/Dataset land for 95% of your pipeline logic. Drop to RDD only when you need element-level Python control (e.g., calling external libraries per-row, custom partitioning logic, or processing truly unstructured byte streams). Convert back to DataFrame as soon as structure is established.

03
Schema Strategies

Schema definition is the most important decision when converting RDDs. Inferred schemas trigger a full RDD scan — never use inference in production pipelines on large datasets.

Python
schema_strategies.py 
from pyspark.sql import SparkSession
from pyspark.sql.types import (
    StructType, StructField, StringType, IntegerType,
    DoubleType, LongType, BooleanType, TimestampType,
    ArrayType, MapType, DateType
)

spark = SparkSession.builder.appName("RDDConversion").getOrCreate()

# ── APPROACH 1: Schema inference (DEV/NOTEBOOK ONLY) ──────────────────────
# Spark samples the entire RDD to figure out types. Never do this in prod!
rdd = spark.sparkContext.parallelize([
    ("alice", 30, 95000.0),
    ("bob",   25, 82000.0),
])
df_inferred = rdd.toDF(["name", "age", "salary"])  # triggers full scan

# ── APPROACH 2: Explicit StructType (PRODUCTION STANDARD) ─────────────────
# No scan. Schema is enforced at parse time. Mismatches raise AnalysisException.
schema = StructType([
    StructField("name",       StringType(),    nullable=False),
    StructField("age",        IntegerType(),   nullable=True),
    StructField("salary",     DoubleType(),    nullable=True),
    StructField("active",     BooleanType(),   nullable=True),
    StructField("created_at", TimestampType(), nullable=True),
])

df = spark.createDataFrame(rdd, schema)

# ── APPROACH 3: DDL string schema (concise, great for SQL practitioners) ───
ddl_schema = "name STRING NOT NULL, age INT, salary DOUBLE, dept STRING"
df_ddl = spark.createDataFrame(rdd, ddl_schema)

# ── APPROACH 4: Nested / complex schema ───────────────────────────────────
complex_schema = StructType([
    StructField("user_id",  LongType(), False),
    StructField("address", StructType([      # Nested struct
        StructField("street", StringType(), True),
        StructField("city",   StringType(), True),
        StructField("zip",    StringType(), True),
    ]), True),
    StructField("tags",    ArrayType(StringType()), True),    # Array
    StructField("scores",  MapType(StringType(), DoubleType()), True),  # Map
])

# ── APPROACH 5: Schema from an existing catalog table (Unity Catalog) ──────
catalog_schema = spark.table("catalog.bronze.events").schema
df_from_catalog = spark.createDataFrame(rdd, catalog_schema)
⚠️
Schema Inference Cost
When you call toDF() or createDataFrame(rdd) without a schema, Spark samples 100% of partitions to determine types. On a 1TB RDD this can add minutes to your job startup. Always define schemas explicitly in production.

Row-Based Conversion

The Row class is the native Spark SQL type. Converting an RDD of Row objects gives you the most control over null handling, nested structures, and field naming.

Python
row_based_conversion.py 
from pyspark.sql import Row

# ── Basic Row creation ─────────────────────────────────────────────────────
raw_rdd = spark.sparkContext.parallelize([
    "alice,30,engineering",
    "bob,25,marketing",
    "carol,,engineering",   # missing age — handle nulls explicitly
])

def parse_csv_to_row(line: str) -> Row:
    parts = line.split(",")
    return Row(
        name   = parts[0].strip(),
        age    = int(parts[1]) if parts[1].strip() else None,
        dept   = parts[2].strip(),
    )

row_rdd = raw_rdd.map(parse_csv_to_row)

schema = StructType([
    StructField("name", StringType(),  False),
    StructField("age",  IntegerType(), True),
    StructField("dept", StringType(),  True),
])

df = spark.createDataFrame(row_rdd, schema)
df.printSchema()
# root
#  |-- name: string (nullable = false)
#  |-- age: integer (nullable = true)
#  |-- dept: string (nullable = true)

# ── Nested Row (struct within struct) ─────────────────────────────────────
Address = Row("street", "city", "zip")
Person  = Row("name", "age", "address")

nested_rdd = spark.sparkContext.parallelize([
    Person("alice", 30, Address("123 Main", "Dallas", "75201")),
    Person("bob",   25, Address("456 Oak",  "Austin", "78701")),
])

nested_schema = StructType([
    StructField("name", StringType(), False),
    StructField("age",  IntegerType(), True),
    StructField("address", StructType([
        StructField("street", StringType(), True),
        StructField("city",   StringType(), True),
        StructField("zip",    StringType(), True),
    ]), True),
])

nested_df = spark.createDataFrame(nested_rdd, nested_schema)
nested_df.select("name", "address.city").show()

Converting Python Named Tuples & Dicts

When your RDD elements are Python dataclasses, named tuples, or dictionaries, you need adapter patterns to safely convert them without runtime type errors.

Python
named_tuples_dicts.py 
from collections import namedtuple
from dataclasses import dataclass, asdict
from pyspark.sql import Row

# ── Pattern 1: namedtuple RDD ──────────────────────────────────────────────
# Spark can directly infer schema from namedtuples (names become columns)
Employee = namedtuple("Employee", ["emp_id", "name", "salary", "dept_id"])

emp_rdd = spark.sparkContext.parallelize([
    Employee(1, "Alice", 95000.0, 10),
    Employee(2, "Bob",   82000.0, 20),
    Employee(3, "Carol", 110000.0, 10),
])

# Best: provide explicit schema even with namedtuples
emp_schema = StructType([
    StructField("emp_id",  LongType(),   False),
    StructField("name",    StringType(), False),
    StructField("salary",  DoubleType(), True),
    StructField("dept_id", IntegerType(), True),
])
emp_df = spark.createDataFrame(emp_rdd, emp_schema)

# ── Pattern 2: dict RDD → Row RDD → DataFrame ─────────────────────────────
dict_rdd = spark.sparkContext.parallelize([
    {"transaction_id": "T001", "amount": 150.75, "currency": "USD"},
    {"transaction_id": "T002", "amount": 99.99,  "currency": "EUR"},
])

# Map dicts → Rows, enforcing known fields (safe against extra/missing keys)
EXPECTED_FIELDS = ["transaction_id", "amount", "currency"]

row_rdd = dict_rdd.map(lambda d: Row(**{k: d.get(k) for k in EXPECTED_FIELDS}))

txn_schema = StructType([
    StructField("transaction_id", StringType(), False),
    StructField("amount",         DoubleType(), True),
    StructField("currency",       StringType(), True),
])
txn_df = spark.createDataFrame(row_rdd, txn_schema)

# ── Pattern 3: dataclass RDD (Python 3.7+) ────────────────────────────────
@dataclass
class SensorReading:
    sensor_id:  str
    temperature: float
    humidity:   float
    ts:         int   # unix epoch ms

sensor_rdd = spark.sparkContext.parallelize([
    SensorReading("S01", 72.4, 65.2, 1714000000000),
    SensorReading("S02", 68.1, 71.8, 1714000001000),
])

# Convert to Row via asdict()
row_rdd2 = sensor_rdd.map(lambda s: Row(**asdict(s)))

sensor_schema = StructType([
    StructField("sensor_id",   StringType(), False),
    StructField("temperature", DoubleType(), True),
    StructField("humidity",    DoubleType(), True),
    StructField("ts",          LongType(),   False),
])
sensor_df = spark.createDataFrame(row_rdd2, sensor_schema)

JSON & Semi-Structured RDDs

A very common real-world pattern: your RDD contains raw JSON strings from Kafka, files, or API responses. Spark's spark.read.json(rdd) and from_json() column function handle these elegantly.

Python
json_rdd_conversion.py 
import json
from pyspark.sql.functions import from_json, col, get_json_object

# ── Method A: spark.read.json(string_rdd) ─────────────────────────────────
# Easiest, but infers schema from sample — OK for exploration
json_rdd = spark.sparkContext.parallelize([
    '{"id":1,"event":"click","user":{"name":"alice","tier":"gold"},"tags":["promo","new"]}',
    '{"id":2,"event":"view","user":{"name":"bob","tier":"silver"},"tags":["organic"]}',
])

df_a = spark.read.json(json_rdd)  # infers nested schema automatically
df_a.printSchema()

# ── Method B: from_json() on a value column (Kafka / Streaming pattern) ────
# Recommended: explicit schema, works on streaming DataFrames too
event_schema = StructType([
    StructField("id",    LongType(),   False),
    StructField("event", StringType(), True),
    StructField("user",  StructType([
        StructField("name", StringType(), True),
        StructField("tier", StringType(), True),
    ]), True),
    StructField("tags", ArrayType(StringType()), True),
])

# Convert raw RDD to single-column DataFrame, then parse
raw_df = json_rdd.toDF(["raw_json"])  # single string column
parsed_df = (
    raw_df
    .withColumn("parsed", from_json(col("raw_json"), event_schema))
    .select("parsed.*")  # flatten the struct
)

# ── Method C: mapPartitions + json.loads (for complex parse logic) ─────────
def parse_partition(records):
    for raw in records:
        try:
            obj = json.loads(raw)
            yield Row(
                id        = obj.get("id"),
                event     = obj.get("event"),
                user_name = obj.get("user", {}).get("name"),
                tags      = obj.get("tags", []),
            )
        except (json.JSONDecodeError, KeyError):
            yield Row(id=None, event="_PARSE_ERROR", user_name=None, tags=[])

flat_schema = StructType([
    StructField("id",        LongType(),              True),
    StructField("event",     StringType(),            True),
    StructField("user_name", StringType(),            True),
    StructField("tags",      ArrayType(StringType()), True),
])
df_c = spark.createDataFrame(json_rdd.mapPartitions(parse_partition), flat_schema)

Full ETL Pipeline Pattern Pipeline

The most common real-world pattern: legacy code ingests binary or text data into an RDD, runs some custom logic, then needs to join with catalog data and write to Delta Lake. Here's the complete pattern.

Source Raw Files / Kafka
→
RDD Stage Parse / Validate
→
DataFrame Transform / Join
→
Sink Delta Lake / Parquet
Python
etl_pipeline_job.py 
from pyspark.sql import SparkSession, Row
from pyspark.sql.functions import col, lit, current_timestamp, to_date
from pyspark.sql.types import *
import re, logging

logger = logging.getLogger(__name__)

# ─────────────────────────────────────────────────────────────────────────────
# STEP 1: Schema definition (define ONCE, reuse everywhere)
# ─────────────────────────────────────────────────────────────────────────────
CLAIM_SCHEMA = StructType([
    StructField("claim_id",       StringType(),  False),
    StructField("patient_id",     LongType(),    False),
    StructField("provider_npi",   StringType(),  True),
    StructField("procedure_code", StringType(),  True),
    StructField("amount",         DoubleType(),  True),
    StructField("service_date",   StringType(),  True),  # parse to date later
    StructField("_parse_errors",  StringType(),  True),  # audit field
])

# ─────────────────────────────────────────────────────────────────────────────
# STEP 2: RDD parsing function — runs on workers, no SparkSession access
# ─────────────────────────────────────────────────────────────────────────────
def parse_claim_line(line: str) -> Row:
    """Parse pipe-delimited claim record. Returns Row with error field."""
    errors = []
    try:
        parts = line.split("|")
        if len(parts) < 6:
            raise ValueError(f"Expected 6 fields, got {len(parts)}")

        claim_id       = parts[0].strip()
        patient_id     = int(parts[1].strip())
        provider_npi   = parts[2].strip() or None
        procedure_code = parts[3].strip() or None
        amount         = float(parts[4].strip())
        service_date   = parts[5].strip() or None

        # Business validations
        if amount < 0:
            errors.append("NEGATIVE_AMOUNT")
        if provider_npi and not re.match(r"^\d{10}$", provider_npi):
            errors.append("INVALID_NPI")

        return Row(
            claim_id=claim_id, patient_id=patient_id,
            provider_npi=provider_npi, procedure_code=procedure_code,
            amount=amount, service_date=service_date,
            _parse_errors=",".join(errors) if errors else None,
        )
    except Exception as e:
        return Row(
            claim_id="_UNPARSEABLE", patient_id=-1,
            provider_npi=None, procedure_code=None,
            amount=0.0, service_date=None,
            _parse_errors=f"PARSE_FAIL:{str(e)[:200]}",
        )

# ─────────────────────────────────────────────────────────────────────────────
# STEP 3: Build the pipeline
# ─────────────────────────────────────────────────────────────────────────────
def run_claims_pipeline(spark: SparkSession, input_path: str, output_path: str):

    # 3a. Ingest as raw text RDD (needed for non-standard delimited files)
    raw_rdd = (
        spark.sparkContext
        .textFile(input_path)
        .filter(lambda line: line.strip() and not line.startswith("#"))  # skip headers/comments
    )

    # 3b. Parse RDD → Row RDD → DataFrame (THE CONVERSION POINT)
    claims_df = (
        spark.createDataFrame(
            raw_rdd.map(parse_claim_line),
            CLAIM_SCHEMA
        )
        .withColumn("service_date", to_date(col("service_date"), "yyyy-MM-dd"))
        .withColumn("_ingested_at", current_timestamp())
        .cache()  # cache — we'll access twice (good + error paths)
    )

    # 3c. Split good records from error records
    good_df  = claims_df.filter(col("_parse_errors").isNull())
    error_df = claims_df.filter(col("_parse_errors").isNotNull())

    # 3d. Enrich with provider lookup from Delta catalog
    provider_df = spark.table("catalog.silver.providers").select(
        "npi", "provider_name", "specialty"
    )

    enriched_df = (
        good_df
        .join(provider_df, good_df.provider_npi == provider_df.npi, "left")
        .drop("npi", "_parse_errors")
    )

    # 3e. Write output — Delta Lake (merge schema safe)
    (
        enriched_df.write
        .format("delta")
        .mode("append")
        .partitionBy("service_date")
        .option("mergeSchema", "true")
        .save(output_path)
    )

    # 3f. Write error quarantine
    if error_df.count() > 0:
        error_df.write.format("delta").mode("append").save(f"{output_path}/_quarantine")
        logger.warning(f"Quarantined {error_df.count()} records")

    claims_df.unpersist()
    return enriched_df

Streaming Pipeline: Kafka → DataFrame Pipeline

In Structured Streaming, the value from Kafka is a byte array — you can't use textFile. The standard pattern converts the Kafka value column using from_json() with an explicit schema, avoiding RDDs entirely for streaming jobs.

💡
RDD vs Structured Streaming
In DStream (legacy streaming), you received an RDD every micro-batch and converted it. In Structured Streaming (the modern approach), you work directly with DataFrames — the conversion happens at the Kafka source level. If you're maintaining DStream code, migration to Structured Streaming is the right move.
Python
kafka_streaming_pipeline.py 
from pyspark.sql.functions import (
    from_json, col, window, sum, count, current_timestamp
)

# ── Schema for Kafka message payload ──────────────────────────────────────
ORDER_SCHEMA = StructType([
    StructField("order_id",   StringType(),  False),
    StructField("user_id",    LongType(),    False),
    StructField("amount",     DoubleType(),  True),
    StructField("status",     StringType(),  True),
    StructField("event_time", TimestampType(), True),
])

# ── Read from Kafka (value is binary → cast to string → parse JSON) ────────
kafka_df = (
    spark.readStream
    .format("kafka")
    .option("kafka.bootstrap.servers", "broker:9092")
    .option("subscribe", "orders-topic")
    .option("startingOffsets", "latest")
    .load()
)

# THE CONVERSION: binary kafka value → typed DataFrame
orders_df = (
    kafka_df
    .select(
        from_json(
            col("value").cast("string"),  # binary → string
            ORDER_SCHEMA
        ).alias("data"),
        col("timestamp").alias("kafka_ts"),
    )
    .select("data.*", "kafka_ts")  # flatten
    .filter(col("order_id").isNotNull())  # drop nulls from bad JSON
)

# ── Windowed aggregation (5-min tumbling window on event_time) ─────────────
agg_df = (
    orders_df
    .withWatermark("event_time", "10 minutes")
    .groupBy(
        window(col("event_time"), "5 minutes"),
        col("status")
    )
    .agg(
        count("*").alias("order_count"),
        sum("amount").alias("total_amount"),
    )
)

# ── Write to Delta Lake (append mode with checkpointing) ───────────────────
query = (
    agg_df.writeStream
    .format("delta")
    .outputMode("append")
    .option("checkpointLocation", "/mnt/checkpoints/orders-agg")
    .trigger(processingTime="30 seconds")
    .start("/mnt/delta/orders_summary")
)
query.awaitTermination()

# ssc = StreamingContext(sc, batchDuration=30)

Delta Lake & Unity Catalog Integration Advanced

In Databricks environments with Unity Catalog, schema can be pulled from the catalog at runtime — making your conversion code self-adapting to schema evolution without code changes.

Python
delta_unity_catalog.py 
from delta.tables import DeltaTable
from pyspark.sql.functions import col, current_timestamp, sha2, concat_ws

# ── Pull schema from Unity Catalog at runtime ─────────────────────────────
TARGET_TABLE = "main.healthcare.claims_bronze"

def get_target_schema(spark, table_name):
    """Fetch schema from catalog; create table if first run."""
    try:
        return spark.table(table_name).schema
    except:
        # Table doesn't exist yet — return bootstrap schema
        return CLAIM_SCHEMA

# ── RDD → DataFrame → MERGE into Delta (upsert pattern) ───────────────────
def merge_claims_to_delta(spark, raw_rdd, table_name):
    schema = get_target_schema(spark, table_name)

    # Convert RDD → DataFrame with catalog schema
    incoming_df = (
        spark.createDataFrame(
            raw_rdd.map(parse_claim_line),
            schema
        )
        .withColumn("_row_hash", sha2(concat_ws("|", col("claim_id"), col("amount")), 256))
        .withColumn("_updated_at", current_timestamp())
    )

    if DeltaTable.isDeltaTable(spark, f"/mnt/delta/{table_name.replace('.','/')}"):
        # MERGE (upsert): update if hash changed, insert if new
        delta_table = DeltaTable.forName(spark, table_name)
        (
            delta_table.alias("target")
            .merge(
                incoming_df.alias("source"),
                "target.claim_id = source.claim_id"
            )
            .whenMatchedUpdate(
                condition="target._row_hash != source._row_hash",
                set={"amount": "source.amount", "_updated_at": "source._updated_at"}
            )
            .whenNotMatchedInsertAll()
            .execute()
        )
    else:
        # First write — create the Delta table
        (
            incoming_df.write
            .format("delta")
            .mode("overwrite")
            .partitionBy("service_date")
            .saveAsTable(table_name)  # registers in Unity Catalog
        )

# ── Z-Order optimization after write ──────────────────────────────────────
spark.sql(f"""
    OPTIMIZE {TARGET_TABLE}
    ZORDER BY (provider_npi, service_date)
""")

Partitioned Batch Ingestion Pattern Pipeline

For high-volume ingestion jobs where RDDs carry partition metadata (date, region, source), you need to inject partition columns during the RDD→DataFrame conversion.

Python
partitioned_ingestion.py 
from pyspark.sql.functions import lit, input_file_name, regexp_extract
from datetime import date

# ── Strategy 1: Inject partition values via mapPartitionsWithIndex ─────────
def tag_with_source(partition_idx: int, records, source_file: str):
    for line in records:
        try:
            row = parse_claim_line(line)
            # Convert Row to dict, add source fields, recreate Row
            d = row.asDict()
            d["_source_file"] = source_file
            d["_partition_id"] = partition_idx
            yield Row(**d)
        except Exception:
            continue

# ── Strategy 2: Process file-per-date with explicit partition column ────────
def ingest_partitioned_files(spark, base_path: str, processing_date: date):
    """Ingest a date partition folder, preserving partition columns."""
    date_str  = processing_date.strftime("%Y-%m-%d")
    file_path = f"{base_path}/dt={date_str}/*.pipe"

    raw_rdd = (
        spark.sparkContext
        .textFile(file_path)
        .filter(lambda l: not l.startswith("#"))
    )

    # Repartition for parallelism (match cluster cores)
    repartitioned = raw_rdd.repartition(200)

    augmented_schema = StructType(CLAIM_SCHEMA.fields + [
        StructField("processing_date", DateType(),    False),
        StructField("source_region",   StringType(), True),
    ])

    # Parse + inject partition values per row
    def parse_and_tag(line):
        row_dict = parse_claim_line(line).asDict()
        row_dict["processing_date"] = processing_date
        row_dict["source_region"]   = "us-east"
        return Row(**row_dict)

    df = spark.createDataFrame(
        repartitioned.map(parse_and_tag),
        augmented_schema
    )

    # Write using partition overwrite (replaces just this date partition)
    spark.conf.set("spark.sql.sources.partitionOverwriteMode", "dynamic")
    (
        df.write
        .format("delta")
        .mode("overwrite")
        .partitionBy("processing_date", "source_region")
        .save("/mnt/delta/claims_bronze")
    )
    return df

UDFs & Pandas UDFs for Complex Transformations Advanced

When your transformation logic is too complex for DataFrame column operations, but you want to stay in DataFrame land (avoiding RDD round-trips), Pandas UDFs (vectorized UDFs) are the high-performance answer. They operate on Pandas Series batches rather than row-by-row.

Python
pandas_udf_patterns.py 
import pandas as pd
from pyspark.sql.functions import pandas_udf
from pyspark.sql.types import DoubleType, StringType

# ── Pattern: Vectorized scalar UDF (Series → Series) ──────────────────────
# 10-100x faster than row-by-row Python UDFs
# Perfect for: ML scoring, regex normalization, numeric computation

@pandas_udf(DoubleType())
def risk_score(amount: pd.Series, age_days: pd.Series) -> pd.Series:
    """Compute fraud risk score vectorized — no Python loop."""
    normalized_amt = (amount - amount.mean()) / (amount.std() + 1e-9)
    recency_factor = pd.Series(age_days).apply(lambda d: 1.0 / (d + 1))
    return (normalized_amt * 0.6 + recency_factor * 0.4).clip(0, 1)

scored_df = claims_df.withColumn(
    "risk_score",
    risk_score(col("amount"), col("age_days"))
)

# ── mapInPandas: full partition → partition transform ──────────────────────
# Use when you need the entire partition as a DataFrame
# Great for: applying sklearn models, external API enrichment per batch

def apply_ml_model(pdf: pd.DataFrame) -> pd.DataFrame:
    import mlflow
    model = mlflow.pyfunc.load_model("models:/fraud_detector/Production")

Performance Pitfalls & Tuning Perf
Pitfall Symptom Fix
Schema inference on large RDD Slow job start, full scan log Always provide explicit StructType
collect() before createDataFrame Driver OOM on large datasets Keep data in RDD, pass RDD directly to createDataFrame
Row-by-row Python UDF Slow executor logs, Python serialization overhead Use Pandas UDF (vectorized) or move to native columns
RDD.map() then createDataFrame without repartition Task skew, slow stragglers Repartition before createDataFrame or after conversion
Frequent RDD↔DataFrame round-trips Repeated serialization cost Do all RDD work first, convert ONCE, stay in DataFrame
No cache() before dual-read Job recomputes the full RDD twice Cache after conversion if the DataFrame is used 2+ times

Anti-Patterns to Avoid
🚫
Never Do These in Production
These patterns look harmless in notebooks but will cause OOM crashes, slow pipelines, or data loss in production cluster jobs.
Python
anti_patterns.py 
# ❌ ANTI-PATTERN 1: collect() before createDataFrame
# collect() brings ALL data to the driver — kills it on large datasets
data = rdd.collect()               # ← NEVER on large RDDs
df  = spark.createDataFrame(data)   # ← driver OOM waiting to happen
# ✅ FIX: pass the RDD directly
df  = spark.createDataFrame(rdd, schema)

# ❌ ANTI-PATTERN 2: SparkSession in map() lambda
# SparkSession is NOT serializable — this crashes the executor
rdd.map(lambda x: spark.createDataFrame([x]))  # ← NotSerializableException
# ✅ FIX: do all DataFrame ops AFTER conversion, not inside map()

# ❌ ANTI-PATTERN 3: Multiple RDD→DataFrame conversions in a loop
dfs = []
for file in files:
    rdd = sc.textFile(file)
    dfs.append(spark.createDataFrame(rdd.map(parse), schema))  # ← creates N jobs
result = reduce(lambda a, b: a.union(b), dfs)

# ✅ FIX: union RDDs first, then convert ONCE
from functools import reduce
combined_rdd = reduce(lambda a, b: a.union(b), [sc.textFile(f) for f in files])
result       = spark.createDataFrame(combined_rdd.map(parse), schema)

# ❌ ANTI-PATTERN 4: Converting to RDD just to use Python built-ins
# Every .rdd access forces Tungsten → Java → Python deserialization
count_by_dept = df.rdd.map(lambda r: (r.dept, 1)).reduceByKey(lambda a, b: a+b)
# ✅ FIX: use DataFrame groupBy — Catalyst optimizes it
count_by_dept = df.groupBy("dept").count()

# ❌ ANTI-PATTERN 5: Schema inference on prod data
df = rdd.toDF()  # ← samples entire dataset; breaks on null columns
# ✅ FIX: always define schema explicitly
df = spark.createDataFrame(rdd, PROD_SCHEMA)

# ❌ ANTI-PATTERN 6: Ignoring parse errors (silent data loss)
df = spark.createDataFrame(rdd.map(lambda x: int(x)), schema)  # crashes on bad int
# ✅ FIX: use try/except in parse functions, write errors to quarantine table

# ════════════════════════════════════════════════════════════════
#   RDD → DATAFRAME CONVERSION CHEATSHEET
#   John's Senior Data Engineer Reference | Apache Spark
# ════════════════════════════════════════════════════════════════

# ── METHOD 1: toDF (notebook / exploration only) ──────────────
df = rdd.toDF(["col_a", "col_b", "col_c"])

# ── METHOD 2: createDataFrame — minimal (infers schema) ───────
df = spark.createDataFrame(rdd)

# ── METHOD 3: createDataFrame — with explicit schema (PROD) ───
schema = StructType([StructField("id", LongType(), False), ...])
df = spark.createDataFrame(rdd, schema)

# ── METHOD 4: createDataFrame — DDL string schema ─────────────
df = spark.createDataFrame(rdd, "id LONG, name STRING, amount DOUBLE")

# ── METHOD 5: Row objects ──────────────────────────────────────
row_rdd = rdd.map(lambda x: Row(id=x[0], name=x[1]))
df = spark.createDataFrame(row_rdd, schema)

# ── METHOD 6: JSON string RDD ──────────────────────────────────
df = spark.read.json(json_rdd)                       # infers schema
df = raw_df.select(from_json(col("val"), schema).alias("d")).select("d.*")

# ── REVERSE: DataFrame → RDD ───────────────────────────────────
rdd         = df.rdd                          # RDD[Row]
string_rdd  = df.rdd.map(lambda r: str(r))   # RDD[str]
tuple_rdd   = df.rdd.map(lambda r: r.asDict()) # RDD[dict]

# ── TYPE REFERENCE ─────────────────────────────────────────────
# StringType()    IntegerType()   LongType()      FloatType()
# DoubleType()    BooleanType()   DateType()      TimestampType()
# ArrayType(T)    MapType(K,V)    StructType([…]) BinaryType()
# ByteType()      ShortType()     DecimalType(p,s) NullType()

# ── KEY RULES (commit these to memory) ────────────────────────
# 1. Always pass explicit schema in production
# 2. Union RDDs before converting (one conversion per pipeline)
# 3. Never use SparkSession inside map()/flatMap()
# 4. Repartition RDD before map() for large datasets
# 5. Cache AFTER conversion if DF used more than once
# 6. Prefer Pandas UDF over Python UDF; prefer DF ops over both
# 7. Drop to RDD ONLY for binary/custom format parsing
# 8. Structured Streaming: use from_json(), not DStream + RDD
✅
Decision Tree Summary
Got structured/semi-structured data? → Start with spark.read.*, never touch an RDD.
Binary or custom delimiter? → sc.binaryFiles() or sc.textFile() → parse to Rows → createDataFrame(rdd, schema) ONCE.
Streaming? → Structured Streaming + from_json(), not DStream.
Complex per-row Python logic after conversion? → Pandas UDF.
Need to write to Delta? → DataFrame → .write.format("delta").partitionBy(...).save() or MERGE.
